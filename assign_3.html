Teaching page for a minimal-background audience

Why regex equivalence can become extremely hard

This page turns the ideas from the source material into a plain-language story. It starts with the basic idea of regular expressions, then explains space complexity, PSPACE, and finally why adding squaring to regexes pushes the equivalence problem to EXPSPACE-complete.

Big picture

regular expressions space complexity PSPACE EXPSPACE

The key idea is that more expressive descriptions can hide much larger structures. Once regexes can use a squaring operation R^2 = RR, deciding whether two expressions describe the same language can require exponential space.

Three sentences to remember

1

Space complexity counts how much memory an algorithm uses, not how long it runs.

2

PSPACE is the class of problems solvable with polynomial memory.

3

Regex equivalence with squaring is so expressive that it reaches EXPSPACE-complete territory.

1. Start with the basics: what is a regular expression?

A regular expression is a compact way to describe a set of strings. For example, a*b means "any number of a's, followed by one b." The important question in the source material is not just whether an expression matches a string, but whether two expressions describe the same set of strings.

Matching

Does one specific string belong to the language?

Equivalence

Do two expressions accept exactly the same strings?

Why squaring matters

The operator R^2 means "concatenate the pattern with itself." That small change makes the comparison problem much harder.

2. What does space complexity mean?

Think of space as the number of sticky notes a computer is allowed to keep on its desk while solving a problem. If it can reuse those notes, it may still take a long time, but it does not need much memory.

Plain version: a problem is in SPACE(f(n)) if some machine can solve inputs of size n using at most O(f(n)) memory cells.

Why this is different from time:

  • Time counts how many steps are taken.
  • Space counts how much memory is needed at once.
  • One machine can reuse the same memory cells again and again.
A

Small space can still mean big time. A depth-first search can walk through a huge tree while only keeping the current path in memory.

B

That is the point of PSPACE. It captures problems that are manageable in memory, even if they take a long time.

3. PSPACE in one sentence

PSPACE is the class of problems solvable using polynomial space. In the source material, this is the big milestone after the basic space-versus-time relationship is introduced.

Useful intuition: polynomial space is memory that grows reasonably with the input size.

PSPACE = union of all SPACE(nk) classes for constant k

Example style from the module:

  • game-tree search
  • logical formulas like TQBF
  • problems with huge branching but manageable recursion depth
The deep lesson is that a problem can look enormous if you draw out every possibility, but still be tame if you only need to store one branch at a time.

4. Why regex equivalence with squaring jumps to EXPSPACE

The source page states the theorem from Stockmeyer and Meyer: once you extend regular expressions with squaring, deciding equivalence becomes EXPSPACE-complete. The audience-friendly version is simple: squaring lets a short expression describe a much larger structure, and the comparison process can explode in size.

What squaring does

R^2 means RR. It is a compact way to repeat a pattern twice.

Why that is powerful

Repeated structure can hide exponential growth in the size of the automaton you would otherwise build.

What EXPSPACE means

It is the class of problems solvable with exponential space, which is far beyond polynomial space.

Intuition, not proof: if the smallest machine that captures the language can be exponentially large, then checking equivalence can force you to reason over that exponential structure.

What the theorem says: the problem is not just hard; it is complete for the hardest problems in EXPSPACE. That makes it a benchmark example in complexity theory.
